Portfolio Containerization Steps

What I did

  • Reviewed the project structure and identified that the Docker setup was invalid for a static portfolio site.
  • Opened the existing Docker file and found that it started with a fortune image and then tried to switch into an Nginx server. That was the root problem.
  • Replaced the broken Dockerfile with one clean Nginx container that serves the portfolio files directly.
  • Updated the Compose files so the site is published to port 8080 on the host machine.
  • Created the missing portfolio assets so there is an actual website to serve, including the main HTML page, the stylesheet, and the filtering script.
  • Updated the README so the run steps match the final portfolio version instead of the earlier catalog version.

Step-by-step: how to do it yourself

  1. Open the folder where your portfolio project is saved.
  2. Make sure the files index.html, styles.css, and script.js are present.
  3. Create or replace the Dockerfile with this content:
    4. FROM nginx:alpine

LABEL Name=buildupporfolio Version=0.0.1

COPY . /usr/share/nginx/html

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
  4. In the Compose file, map port 8080 to container port 80.
  5. Build the image with:
    6. docker build -t buildupporfolio .
  6. Run the container with:
    7. docker run -p 8080:80 buildupporfolio
  7. Open http://localhost:8080 in your browser.

What you should do next

  1. Open a terminal in the project folder.
  2. Make sure Docker Desktop is running on your computer.
  3. Run the image build command:
    4. docker build -t buildupporfolio .
  4. Run the container:
    5. docker run -p 8080:80 buildupporfolio
  5. Open your browser and go to http://localhost:8080.
  6. If the build fails, restart Docker Desktop and retry. If the issue continues, check that the Docker engine is running properly.
